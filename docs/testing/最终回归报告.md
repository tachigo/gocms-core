## ❌ 失败用例分析（3 个）

### FAIL-1: test_permission.py::TestRBACMiddlewareP0::test_author_create_taxonomy

- **预期：** author 创建 taxonomy term → HTTP 403
- **实际：** HTTP 200，成功创建（`{"code":0,"data":{"id":1}}`）
- **根因：** author 角色被种子数据赋予了 `taxonomy:create:own` 权限。RBAC 中间件检查 hasPerm=true 后放行，但 taxonomy 无 owner 概念，scope:own 的 create 操作等同于允许。
- **分类：** ⚠️ **已有权限策略 Bug**（非本次修复引入）
- **建议：** author 不应被赋予 taxonomy/menu 的 write 权限，或中间件对无 owner 资源的 scope:own create 应拒绝。

### FAIL-2: test_page.py::TestPageAdminP0::test_editor_delete_others

- **预期：** editor 删除他人的 page → HTTP 403
- **实际：** HTTP 200，成功删除（`{"code":0,"data":{}}`）
- **根因：** editor 角色种子数据配置为 `page:delete:all`（scope:all），因此可删除任何人的 page。
- **分类：** ⚠️ **权限策略设计争议**（非本次修复引入）
- **建议：** 如需限制 editor 只能删自己的 page，需将 editor 的 page delete scope 改为 own。

### FAIL-3: test_taxonomy.py::TestTaxonomyAdminP0::test_author_create_denied

- **预期：** author 创建 taxonomy term → HTTP 403
- **实际：** HTTP 200，成功创建（`{"code":0,"data":{"id":7}}`）
- **根因：** 同 FAIL-1，重复测试点。
- **分类：** ⚠️ **已有权限策略 Bug**

> **3 个 Fail 均为已有的 RBAC 权限策略 Bug，非本次 scope:own 修复引入的回归。**
