### ❌ 失败项（1 条 — 真 Bug）

| ID | 场景 | 问题 | 根因 |
|----|------|------|------|
| **FE-051** | 术语列表 | `GET /api/admin/taxonomies/article_category/terms` 返回 code=51 | **路由参数类型错误**: vocabId 被当成整数校验（`must be equal or greater than 1`），但词汇表 ID 是字符串（如 `article_category`） |

**影响：** 前端无法通过字符串 ID 获取术语列表，影响分类管理功能。

**临时方案：** 用数字 ID（如 `vocabId=1` 或 `2`）可以正常获取术语。

**建议修复：** 将 vocabId 参数类型改为 string，或同时支持 string/number。

### ⏭️ 路径变更项（非 Bug，但需与 PRD 同步）

| PRD 路径 | 实际路径 | 说明 |
|----------|----------|------|
| `/api/users` | `/api/admin/users` | 管理用户列表需 admin 前缀 |
| `/api/roles` | `/api/admin/roles` | 角色列表需 admin 前缀 |
| `/api/permissions/available` | `/api/admin/permissions/available` | 权限矩阵需 admin 前缀 |
| `/api/settings/admin` | `/api/admin/settings` | 管理设置路径变更 |

**说明：** 实际路径统一在 `/api/admin/*` 下，符合"后台管理 API 需认证"的设计。PRD 文档需更新。
